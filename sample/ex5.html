Exercise 5 - Recursion

Task 1 - Recursion on Linked Lists

In this task, you'll return to a familiar linked list function, but implement it using recursion rather than using a loop.

Learning goals:

  • Practice calling functions recursively
  • Use the recursive structure of linked lists

Starter code:

  • list_map.py
  • list_map_test.py

File to submit:

  • list_map.py

In the starter code, write a function map_f(linked_list, f), where f is a function, which returns a new recursive linked list whose items are obtained by applying f to all of the items in linked_list. Note that linked_list is a recursive linked list, which is why there's an import statement.

Your function should not mutate the original list, and must be recursive! No loops allowed!

Task 2 - A Family Tree

In this task, you'll consider a more complicated type of recursive structure: a family tree.

Learning goal:

  • Apply recursion to a tree-like data structure

Starter code:

  • family.py
  • family_test.py

File to submit:

  • family.py

Consider the following situation. A person has a name and some (or no) children, each of whom is another person with a name and some (or no) children. This is modelled in the starter code using a basic Person class with two attributes, name and children. Note that the children attribute is just a plain old Python list of Person objects, not a linked list; a person with no children would have an empty list stored in this attribute.

A descendant of a Person is either a child of that Person, or a child of a child of that Person, etc. So clearly, a Person with no children has no descendants. Complete the body of the method count_descendants, which returns the number of descendants of self.

The million dollar recursive question: How is the number of descendants of a person related to the number of descendants of his/her children?
