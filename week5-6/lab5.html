Lab 5 - Recursion Explosion

Task 1: Thinking Recursively (60 min)

Learning goals:

  • Analyse a problem recursively
  • Translate recursive understanding into a recursive function

In this task, you will get lots of practice with thinking recursively. Each of the problems below has a recursive structure that you can use to write extremely simple code.

Even more important than before, it is vital that you design with pencil and paper before writing any code! Before even touching the keyboard, make sure you can answer the following two questions:

  • What are the base cases? That is, what are the simplest or smallest problem sizes that you can solve directly?
  • How can you relate the problem of size n to the same problem structure of a smaller size?

The second question really is the heart of thinking recursively. We've provided some hints and guidelines to get you started on the questions below.

Reversing a String

You are given as input a string, and want to output the reverse of that string. For example, for the input string 'computer', your function should output 'retupmoc'.

Hint: take a look at 'computer'. Remove the first letter to get 'omputer'. How does the reversal of the rest of the string related to the reversal of the original?

Once you're ready, open up the starter code and implement your function. A basic test file is provided.

Subsets

You are given as input a list with no repetitions. You would like to output a list of lists that represents all of the different subsets of numbers from the original list. A subset is a list of numbers that appeared in the original list. Order doesn't matter, so the subsets [1,2] and [2,1] are considered identical. The empty list is considered to be a subset of any list; also, the whole list is also considered to be a subset of itself.

For example, the input list [1,2,3] has 8 subsets:

[
      [], 
      [1], 
      [2], 
      [3], 
      [1, 2], 
      [1, 3], 
      [2, 3], 
      [1, 2, 3]
    ]

Hint 1: This should remind you of the binary string example from lecture.

Hint 2: First, try writing down all of the subsets of [1,2,3]. You should have 8. Divide these subsets into two groups: those that contain 1, and those that don't. What do you notice?

Task 2: Recursive Linked Lists (40 min)

Learning goals:

  • Understand the recursive implementation of a linked list
  • Write both non-mutating and mutating recursive methods for a linked list

In this task, you will again practice thinking recursively - this time with the familiar linked list. In the starter code, you are provided with all of the functions from lecture. Complete the methods __contains__ and insert according to their docstrings, using recursion in each case. __contains__ is the easiest because it doesn't involve mutation, so start there and take your time to make sure you really understand recursion on linked lists.

Note that once you get the hang of recursion, each of these functions takes no more than 5 lines of code! Remember the recursive structure of linked lists, and structure your functions in the same way!

Task 3: Visualising References (20 min)

Learning goals:

  • Understand how objects are passed as arguments in Python
  • Understand how this often leads to surprising behaviour, so be careful!

Consider the following alternate implementations of insert_first for LinkedListRec:

def insert_first_1(self, item):
        self.rest = self
        self.first = item
    
    def insert_first_2(self, item):
        self.rest = self.first
        self.rest.rest = self.rest
        self.first = item

These don't work - why? Step through with the Python visualiser to see what goes wrong. This is a pretty technical point, but will come back at some time in your career if you don't understand it!

Task 4: Optional problems

If you finish with the problems above, here are some other ones to try. These are certainly not required for completing the lab.

  • Write a function bin_rep(n) that returns a string with the binary representation of n.
  • Extend bin_rep(n) to create base_rep(n, k) to give a string representing non-netavie integer n in base k, where 2 <= k <= 10. In base k, the only digits allowed are {0, ..., k-1}.
  • For a list of distinct integers, L, define switches(L) as the number of pairs in L that are not in increasing order. For example, switches([6, 1, 4]) returns 2, since (6, 1) and (6, 4) are out of order.
